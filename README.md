# نظام الفواتير والمخازن المتكامل (Invoicee System API)

هذا المشروع هو الواجهة الخلفية (API) المتطورة لنظام إدارة المخازن والفواتير. يوفر حلولاً متكاملة لإدارة العمليات التجارية اليومية، من تتبع المخزون إلى إدارة الاشتراكات والتدقيق المالي.

## 🚀 الميزات الرئيسية

### 📦 إدارة المخزون والمنتجات
- **تحكم كامل (CRUD):** إضافة، تعديل، وحذف المنتجات بتفاصيل دقيقة.
- **تتبع الكميات:** تحديث تلقائي للمخزون مع كل عملية بيع أو شراء.
- **تنبيهات نقص المخزون:** نظام ذكي لتنبيه المستخدم عند وصول المنتج للحد الأدنى.
- **البحث المتقدم:** إمكانية البحث بالكود، الاسم، أو الفئة.

### 💰 العمليات المالية والتجارية
- **المبيعات والمشتريات:** تسجيل فواتير البيع والشراء وتتبع تاريخ العمليات.
- **إدارة المصروفات:** تتبع المصروفات اليومية والتشغيلية للمؤسسة.
- **التقارير المالية:** حساب الأرباح والخسائر، وعرض إحصائيات الأداء من خلال لوحة تحكم بيانية.

### 🎫 إدارة الاشتراكات والخطط
- **نظام الخطط المتعددة:** دعم لخطط (تجريبية، أساسية، احترافية).
- **إدارة المدفوعات:** تتبع عمليات الدفع (مثل فودافون كاش) مع نظام تأكيد يدوي من الإدارة.
- **التفعيل التلقائي:** تفعيل الميزات بناءً على صلاحيات الخطة النشطة.

### 🛡️ أدوات الإشراف (SuperAdmin Power Tools)
- **لوحة تحكم إدارية:** مراقبة إحصائيات النظام بالكامل والمستخدمين النشطين.
- **مراجعة العمليات:** نظام قبول أو رفض عمليات الدفع يدوياً وتفعيل الاشتراكات.
- **سجلات التدقيق (Audit Logs):** تتبع تفصيلي لكل الإجراءات الحساسة في النظام لضمان الأمان.
- **إدارة المستخدمين:** تجميد، حذف، أو تحديث صلاحيات المستخدمين.

## 🛠️ التقنيات المستخدمة

- **المحرك الأساسي:** Node.js & Express.js (v5.1.0)
- **قاعدة البيانات:** MongoDB مع Mongoose للنمذجة.
- **الأمان:** 
    - JWT للمصادقة.
    - Bcryptjs لتشفير البيانات.
    - Helmet & HPP و Rate Limiting للحماية من الهجمات.
- **التكامل والبيانات:**
    - XLSX لإدارة تصدير واستيراد ملفات Excel.
    - Axios للطلبات الخارجية.
- **المراقبة والأداء:**
    - Integration مع Sentry لتتبع الأخطاء في الوقت الفعلي.
    - Compression لضغط الطلبات وتحسين السرعة.

## ⚙️ الإعداد والتشغيل

1. **تثبيت التبعيات:**
   ```bash
   npm install
   ```

2. **متغيرات البيئة:**
   قم بإنشاء ملف `.env` وأضف:
   ```env
   PORT=5000
   MONGO_URI=your_mongodb_uri
   JWT_SECRET=your_secret_key
   SENTRY_DSN=your_sentry_dsn
   NODE_ENV=development
   ```

3. **التشغيل:**
   - للتطوير: `npm run dev`
   - للفحص: `npm test`

## 🌐 النشر (Deployment)
المشروع مهيأ تماماً للنشر على منصة **Vercel** باستخدام نظام Serverless Functions، مع وجود ملف `vercel.json` مُعد مسبقاً لضمان أفضل أداء.
